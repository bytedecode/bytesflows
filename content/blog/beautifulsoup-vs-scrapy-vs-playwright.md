## BeautifulSoup vs Scrapy vs Playwright

- **Beautiful Soup** — A **parser** for HTML/XML. You bring the HTML (e.g. via Requests). Best for **static** pages and quick scripts. [Python Web Scraping Guide](/en/blog/python-web-scraping-guide) and [Using Requests](/en/blog/python-web-scraping-guide). Add [residential proxies](/en/blog/residential-proxies) when scaling; [best proxies for web scraping](/en/blog/best-proxies-for-web-scraping).
- **Scrapy** — A **framework** for crawlers: spiders, scheduling, pipelines. Best for **site-wide** crawls and [distributed crawlers](/en/blog/scaling-scrapers-distributed-systems). [Scrapy Framework Guide](/en/blog/scrapy-framework-guide). Use [rotating proxies](/en/blog/rotating-proxies-web-scraping) and [Python proxy scraping](/en/blog/python-scraping-proxy).
- **Playwright** — **Browser automation**: real Chromium/Firefox. Best for **JavaScript-rendered** pages and [scraping dynamic websites](/en/blog/scraping-dynamic-websites-playwright). [Playwright Web Scraping Tutorial](/en/blog/playwright-web-scraping-tutorial) and [Playwright vs Puppeteer](/en/blog/playwright-vs-puppeteer). [Using Proxies with Playwright](/en/blog/using-proxies-playwright) and [residential proxies](/en/blog/residential-proxies).

## When to Use Which

| Need | Tool |
|------|------|
| Static HTML, few pages | Beautiful Soup + Requests |
| Large crawl, many URLs | Scrapy |
| JS-rendered, SPAs, anti-bot | Playwright |
