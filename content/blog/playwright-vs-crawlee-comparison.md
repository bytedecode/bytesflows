## Playwright vs Crawlee

**Playwright** is a **browser automation** library (Chromium, Firefox, WebKit). **Crawlee** is a **scraping/crawling framework** that can use Playwright (or Puppeteer) under the hood and adds **queues**, **storage**, **retries**, and **concurrency**. So: Playwright = browser control; Crawlee = scraping app structure on top. See [Playwright Web Scraping Tutorial](/en/blog/playwright-web-scraping-tutorial) and [Crawlee Web Scraping Tutorial](/en/blog/crawlee-web-scraping-tutorial). For scale, use [residential proxies](/en/blog/residential-proxies) and [best proxies for web scraping](/en/blog/best-proxies-for-web-scraping).

## When to Use Which

- **Playwright alone** — Focused scraping, a few flows, full control. [Scraping dynamic websites](/en/blog/scraping-dynamic-websites-playwright), [using proxies with Playwright](/en/blog/using-proxies-playwright).
- **Crawlee** — Large crawls, URL queues, persistence, built-in best practices. [Building scrapers with Crawlee](/en/blog/crawlee-web-scraping-tutorial). [Playwright vs Puppeteer](/en/blog/playwright-vs-puppeteer) for engine choice inside Crawlee.
